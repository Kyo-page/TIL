# TIL（Today I Learned）の特徴とメリット

## TILとは
TIL（Today I Learned）は、「今日学んだこと」を短く記録するための習慣・リポジトリ形式です。  
主にMarkdownで、1日1トピックを簡潔にまとめます。

---

## 特徴

### 1. 短くてよい
- 数行〜数十行程度でOK
- 完璧なまとめや解説は不要
- 「わかったこと」「つまずいた点」だけでも成立する

### 2. 継続しやすい
- 学習量が少ない日でも書ける
- コードがなくてもコミット可能
- 毎日更新しやすく、習慣化しやすい

### 3. 形式がシンプル
- Markdownファイル1枚から始められる
- フォルダ分けやルールも最低限でよい
- 自分ルールで自由に運用できる

### 4. 内容の幅が広い
- プログラミング（React / JavaScript / CSS など）
- エラーの対処方法
- 環境構築メモ
- 学習中の気づき・考え方

---

## メリット

### 1. GitHubの草が自然に生える
- 無理にコードを書かなくてもコミットできる
- 「草目的」に見えにくく、健全な活動履歴になる

### 2. 学習内容が定着しやすい
- 「書くことで理解が整理される」
- 翌日・数週間後に見返して復習できる
- 同じミスを繰り返しにくくなる

### 3. 成長の可視化ができる
- 過去のTILを見ると理解の深まりがわかる
- 初心者→慣れてきた過程がそのまま残る
- モチベーション維持につながる

### 4. ポートフォリオの補助になる
- 学習姿勢・継続力のアピールになる
- READMEを整えれば公開用にも使える
- 「学び続けている人」という印象を与えやすい

---

## 向いている人
- プログラミング学習を始めたばかりの人
- 毎日少しずつでも前進したい人
- アウトプット習慣を作りたい人
- GitHubを育てたい人

---

## まとめ
TILは  
**「完璧を求めず、毎日少し学びを残すための仕組み」**。  
初心者でも無理なく続けられ、学習・記録・GitHub活用を同時に叶えられる方法です。
