## 1/23

useEffect でのデータフェッチについて学んだ。

### 1. useEffect によるデータフェッチとクリーンアップ
`useEffect` で API 通信を行う際は、メモリリークや意図しない挙動を防ぐための「お掃除（クリーンアップ）」が必須。

- **クリーンアップ関数**: コンポーネントが消える際に `active = false` とすることで、未完了の通信結果が破棄されたコンポーネントの状態を更新するのを防ぐ。
- **Promise の 3 ルート**:
  - `.then()`: 成功時の処理（データのセット）。
  - `.catch()`: 失敗時の処理（エラー情報の保持）。
  - `.finally()`: 共通処理（ローディングフラグのオフ）。

### 2. ユーザー体験を向上させる条件付きレンダリング
三項演算子を使用して、通信状態に応じた UI を提供する。
- **Loading**: スピナーを表示し、処理中であることを伝える。
- **Error**: エラー内容を表示し、「再試行」ボタンを提供することで離脱を防ぐ。
- **Success**: 正常に取得したデータを安全（`?.` オプショナルチェーン）に表示する。

### 3. 実務におけるデータフェッチのあり方
`useEffect` でのフェッチは学習用としては良いが、実務では以下の理由から **React Query (TanStack Query)** や **SWR** などのライブラリが推奨される。
- **キャッシュ管理**: 一度取得したデータを再利用できる。
- **レースコンディション防止**: 古いリクエストが新しいデータを上書きするのを自動で防ぐ。
- **ボイラープレートの削減**: `loading` や `error` の管理をライブラリに一任できる。
